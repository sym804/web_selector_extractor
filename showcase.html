// Features

핵심 기능

QA 엔지니어를 위해 설계된 정밀한 셀렉터 추출과 코드 생성 기능

클릭 기반 셀렉터 추출

페이지의 어떤 엘리먼트든 클릭하면 즉시 7가지 타입의 셀렉터를 추출합니다. id, data-testid, class, CSS, XPath, role, text 셀렉터를 한 번에 확인할 수 있습니다.

유니크 셀렉터 우선 정렬

페이지에서 1개만 매치되는 유니크 셀렉터를 최상위로 추천합니다. UNIQUE 뱃지(초록)와 매치 수 뱃지(노랑)로 각 셀렉터의 정확도를 직관적으로 확인할 수 있습니다.

말풍선 팝업 UI

클릭한 엘리먼트 옆에 말풍선 형태로 셀렉터 목록이 표시됩니다. 페이지 컨텍스트를 벗어나지 않고 바로 원하는 셀렉터를 선택하고 복사할 수 있습니다.

사이드 패널 히스토리

추출한 모든 셀렉터를 사이드 패널에 히스토리로 기록합니다. 여러 엘리먼트의 셀렉터를 순차적으로 추출하고 한꺼번에 관리할 수 있습니다.

Playwright 코드 자동 생성

추출된 셀렉터로 locator, click, fill, assert 4가지 Playwright 코드 스니펫을 자동 생성합니다. 복사 버튼 한 번으로 테스트 코드에 바로 활용할 수 있습니다.

내보내기 (JSON / CSV)

추출된 셀렉터 히스토리를 JSON 또는 CSV 형식으로 내보낼 수 있습니다. 테스트 문서 작성이나 팀 공유, 자동화 파이프라인 연동에 활용하세요.

셀렉터 우선순위

가장 안정적이고 유지보수 가능한 셀렉터를 최우선으로 추천합니다

#1
id 고유 식별자 - 가장 안정적인 셀렉터
#2
data-testid 테스트 전용 속성 - QA 의도를 명확히 표현
#3
unique class 유니크 클래스 - 단일 매치 보장
#4
css CSS 셀렉터 - 구조 기반 탐색
#5
xpath XPath - DOM 트리 완전 탐색
#6
role ARIA 역할 - 접근성 기반 선택
#7
text 텍스트 컨텐츠 - 표시 텍스트 기반
// How It Works

사용법

3단계로 완성하는 QA 셀렉터 추출 워크플로

1

확장 프로그램 활성화

크롬 툴바에서 Selector Extractor 아이콘을 클릭하여 추출 모드를 활성화합니다

2

엘리먼트 클릭

추출하고 싶은 웹 엘리먼트를 클릭하면 말풍선에 셀렉터 목록이 표시됩니다

3

복사 & 활용

원하는 셀렉터를 복사하거나, Playwright 코드를 생성하고, JSON/CSV로 내보냅니다

// Playwright Snippets

코드 자동 생성

추출된 셀렉터로 바로 사용할 수 있는 Playwright 코드를 생성합니다

playwright-test.ts
// locator - 엘리먼트 선택
const element = page.locator('[data-testid="submit-btn"]');

// click - 클릭 액션
await page.locator('[data-testid="submit-btn"]').click();

// fill - 입력 액션
await page.locator('[data-testid="submit-btn"]').fill('텍스트');

// assert - 가시성 검증
await expect(page.locator('[data-testid="submit-btn"]')).toBeVisible();
